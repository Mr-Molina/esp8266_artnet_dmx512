# ESP8266 Microcontroller Hardware Context & Pinout Reference
**Target Audience:** AI Agents, Code Generators, Hardware Compilers
**Purpose:** Single source of truth for ESP8266 (ESP-12E / NodeMCU / Wemos D1 Mini) GPIO capabilities, constraints, and peripheral mapping.

## 1. CORE DIRECTIVES FOR AI AGENTS
When generating wiring diagrams, C++/MicroPython code, or hardware advice for ESP8266, you MUST obey these rules:
* **LOGIC LEVELS:** The ESP8266 is strictly **3.3V logic**. Applying 5V to any digital GPIO will damage the chip.
* **ADC VOLTAGE LIMITS:** The bare ESP8266 chip ADC0 (A0) pin max voltage is **1.0V**. Development boards (NodeMCU, Wemos D1 Mini) include a voltage divider that increases this limit to **3.3V**. Always clarify the hardware form factor before using the ADC.
* **D-PIN TO GPIO MAPPING:** The silkscreen labels (D0, D1, etc.) on development boards do **NOT** match the internal GPIO numbers. Code must address the correct GPIO number or rely on board-specific macro definitions.
* **SPI FLASH PINS:** Never assign GPIOs 6, 7, 8, 9, 10, or 11. They are internally connected to the integrated SPI flash chip. Using them will crash the board.
* **WAKE FROM DEEP SLEEP:** Deep sleep wake-up is strictly tied to **GPIO16** (D0). It must be physically tied to the `RST` pin for the timer wake-up to function.

## 3. PERIPHERAL MAPPING

### 3.1 I2C (Inter-Integrated Circuit)
* **Implementation:** The ESP8266 does not have hardware I2C; it is implemented in software.
* **Default SDA:** GPIO 4 (D2)
* **Default SCL:** GPIO 5 (D1)
* *Note: Any capable GPIO can be configured for I2C in software.*

### 3.2 SPI (Serial Peripheral Interface)
* **Hardware Pins:**
  * **MISO:** GPIO 12 (D6)
  * **MOSI:** GPIO 13 (D7)
  * **SCLK:** GPIO 14 (D5)
  * **CS:** GPIO 15 (D8)

### 3.3 PWM (Pulse Width Modulation)
* **Implementation:** Software PWM is supported on all standard I/O pins (GPIO 0 to GPIO 15).
* **Resolution:** 10-bit (0-1023).
* **Exception:** GPIO 16 (D0) does not support PWM.

### 3.4 Interrupts
* **Availability:** Supported on any GPIO **except GPIO 16** (D0). 
* **Constraint:** GPIO 16 cannot be configured with `INPUT_PULLUP` or hardware interrupts.

### 3.5 Analog to Digital Converter (ADC)
* **Channels:** 1 channel only (ADC0 / A0).
* **Resolution:** 10-bit (0-1023).

## 4. BOOT & STRAPPING PIN BEHAVIORS

The ESP8266 determines its boot mode by reading the logic level of specific pins at startup. Incorrectly loading these pins will prevent the execution of code or brick the flashing process.

### 4.1 Critical Boot Constraints (Strapping Pins)
| GPIO | Required State at Boot | Consequence of Failure |
| :--- | :--- | :--- |
| **0 (D3)** | HIGH | Boot failure (Enters Flashing/Bootloader mode if LOW) |
| **1 (TX)** | HIGH | Boot failure if pulled LOW |
| **2 (D4)** | HIGH | Boot failure if pulled LOW |
| **15 (D8)**| LOW | Boot failure if pulled HIGH |

### 4.2 State at Boot / Reset
When connected to sensitive peripherals like relays or motors, the sudden voltage output during a reset can cause unexpected actuation.
* **Outputs HIGH at Boot:** GPIO 1 (TX), GPIO 3 (RX), GPIO 9, GPIO 10, GPIO 16 (D0).
* **Safe Pins for Relays:** GPIO 4 (D2) and GPIO 5 (D1) are the most stable during the boot sequence and are highly recommended for triggering relays or transistors.

## 5. HARDWARE MISCELLANEOUS
* **Built-in LED:** Usually connected to GPIO 2 (D4). Operates on inverted logic (LOW turns LED ON, HIGH turns LED OFF).
* **RST Pin:** When pulled LOW, the ESP8266 hardware resets.
